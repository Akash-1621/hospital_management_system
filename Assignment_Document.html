Module Description

The Patient Management module is the core module of the CareSphere Hospital Management System. It provides a complete CRUD (Create, Read, Update, Delete) API for managing patient records in a hospital. Built using the MERN Stack (MongoDB, Express.js, React.js, Node.js), this module handles patient registration, record retrieval, updates, and discharge/deletion workflows.

Reports

The module supports data retrieval useful for generating reports:

  • Filtering by date range (startDate, endDate query params) — enables weekly/monthly admission summaries
  • Filtering by department (CARDIOLOGY, NEUROLOGY, ORTHOPEDICS, PEDIATRICS, GENERAL) — enables department-wise patient breakdowns
  • Filtering by status (ADMITTED, DISCHARGED, UNDER_TREATMENT, CRITICAL) — enables status-based tracking reports
  • Search by name/email (search query param) — enables quick patient lookup
  • Pagination (page, limit) — supports large dataset reporting with a default of 50 items per page

Patient Schema

{ name: String (required), age: Number (required, 0-150), gender: String (enum: MALE, FEMALE, OTHER), contact: String (required), email: String, address: String, department: String (enum: CARDIOLOGY, NEUROLOGY, ORTHOPEDICS, PEDIATRICS, GENERAL), medicalHistory: String, admissionDate: Date (default: Date.now), status: String (enum: ADMITTED, DISCHARGED, UNDER_TREATMENT, CRITICAL), assignedDoctor: String, timestamps: true (createdAt, updatedAt auto-generated) }

API Endpoints

# Method Endpoint Description
1 POST /api/patients Create a new patient record
2 GET /api/patients Get all patients (with filters & pagination)
3 GET /api/patients/:id Get a single patient by ID
4 PUT /api/patients/:id Update a patient (partial update supported)
5 DELETE /api/patients/:id Delete a single patient by ID

How to Run

Prerequisites

  • Node.js (v18 or above)
  • MongoDB (running locally on port 27017)

Steps

1 Install frontend dependencies npm install

6 Open in browser http://localhost:5173
